# Shape Browser — Shape ID Documentation
Version: shape-browser-basic

---

## 1. Definition of Shape ID

`Shape.id` is not created by the shape-browser tool.

It is the primary key of the `shapes` table in the MariaDB database,
generated by `exportshapes`.

Simplified database schema:

TABLE shapes (
    id INT PRIMARY KEY,
    parent_id INT,
    dictionary_id INT,
    width INT,
    height INT,
    bits LONGBLOB
)

Therefore:

- Shape ID is assigned during export.
- It uniquely identifies a canonical glyph shape.
- It is stable within a given database.

---

## 2. How Shape ID Enters the Program

In `repository.py`:

SELECT s.id, s.parent_id, s.width, s.height, s.bits
FROM shapes s
JOIN dictionaries d ON s.dictionary_id = d.id
WHERE d.document_id = %s

Each database row contains:

row["id"]

This becomes:

shape = Shape(
    shape_id=row["id"],
    ...
)

Inside `ShapeModel`:

self.shapes[shape.id] = shape

Internally:

model.shapes : Dict[int, Shape]

Shape ID is:

- The dictionary key
- The canonical identifier
- Used for lookup throughout the application

---

## 3. How Shape ID Is Used in the GUI

When inserting into the Treeview:

label = f"[{shape.depth}] Shape {shape.id} (h={shape.height})"

When a node is selected:

text = self.tree.item(node, "text")
shape_id = int(text.split("Shape ")[1].split()[0])
shape = self.model.get_shape(shape_id)

Thus:

- Shape ID is embedded in the display text.
- It is parsed back from the label.
- It is used to retrieve the Shape object.

Current limitation:

Parsing the ID from text is fragile.
A future improvement should store the ID in Treeview metadata:

self.tree.insert(..., values=(shape.id,))
shape_id = self.tree.item(node)["values"][0]

---

## 4. Semantic Meaning of Shape ID

Shape ID identifies:

A canonical glyph bitmap stored in the database.

Important distinction:

Shape ID identifies the glyph pattern.
Occurrences (blits) identify placements of that glyph on pages.

Conceptually:

Shape (canonical glyph)
    ↓
Multiple Occurrences (placements on pages)

Shape ID ≠ occurrence.

---

## 5. Relationship to Hierarchy

Hierarchy is defined by:

parent_id

If:

parent_id == -1 or None

→ The shape is a root.

Otherwise:

shape.parent_id → parent shape.id

Depth is computed dynamically in `ShapeModel`.

Shape ID itself does NOT encode hierarchy information.

---

## 6. Stability and Limitations

Shape IDs:

- Reflect export order.
- Do not encode:
  - visual similarity
  - frequency
  - rarity
  - semantic meaning
  - depth

They are purely technical identifiers.

---

## 7. Summary

Shape ID:

- Comes directly from the database.
- Is the primary key of `shapes`.
- Identifies canonical glyph bitmaps.
- Is used as dictionary key in memory.
- Is currently parsed from GUI labels (basic version).
- Has no semantic meaning beyond identity.
